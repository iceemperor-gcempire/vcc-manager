# VCC Manager — Design Brief

> 디자인 에이전트 / 디자이너에게 전달할 입력 자료. 산출물: 디자인 시스템 토큰 + 핵심 페이지 mockup. 첫 라운드는 토큰 + 3개 핵심 페이지 우선.

---

## 1. 제품 소개

**VCC Manager** 는 다양한 AI 백엔드 (ComfyUI / OpenAI / Gemini) 위에 **이미지·텍스트·영상 생성 워크플로우를 관리** 하는 internal tool 입니다.

- 사용자: 콘텐츠 제작자 (AI 생성 워크플로우를 반복 활용)
- 사용 빈도: 매일 1~수회. 한 세션이 길어질 수 있음 (한 프로젝트 작업 중)
- 사용 환경: 데스크탑이 주 (정밀 작업, 빌더), 모바일은 진행 확인 / 가벼운 편집
- 현재 운영 환경: alpha-vccm-svc.gcempire.net (단일 인스턴스, 다중 사용자)

## 2. 핵심 도메인 개념

| 개념 | 설명 |
|---|---|
| **Server** | ComfyUI / OpenAI / Gemini / OpenAI Compatible 백엔드 — admin 이 등록. |
| **Workboard** | 특정 server 와 outputFormat (image / text / video) 조합. admin 이 입력 양식 (customField) 정의. 사용자는 양식 채워 실행. |
| **Project** | 사용자 워크스페이스. 소속 작업판 + 문서 (UploadedText) + 파이프라인 + 결과물 (이미지/대화) 보유. |
| **문서 (UploadedText)** | 프로젝트 내 텍스트 자산. 태그로 분류 (세계관 / 시스템 프롬프트 / 자유 카테고리). 파이프라인이 가져다 LLM 컨텍스트로 주입. |
| **Pipeline** | 작업판 A → B → C 직선 시퀀스. 단계마다 사전 입력 + 사전 컨텍스트 / 시스템 프롬프트 문서 연결. 백엔드 백그라운드 실행 + 부분 retry. |
| **PipelineRun** | Pipeline 의 실제 실행 인스턴스 (영속). 단계별 상태 / 결과 보존. |

## 3. 핵심 사용자 여정

### 여정 A. 프로젝트 셋업 → 파이프라인 실행 (가장 자주, 시간 가장 많이 보냄)

1. 사용자 로그인 → 프로젝트 목록
2. 새 프로젝트 생성 / 기존 프로젝트 선택
3. 프로젝트 상세 → **세계관 탭** — 세계관 / 시스템 프롬프트 문서 작성 (등장인물 / 톤 / LLM 역할 정의)
4. **파이프라인 탭** — 새 파이프라인 생성 → 단계별로 작업판 추가, 사전 입력 / 자동 주입 / 문서 연결 / 메모 설정
5. **실행** — 초기 프롬프트 입력 → 시작 → 단계별 진행 표시
6. 페이지 떠나 다른 작업 → **파이프라인 히스토리 탭** 에서 결과 확인 / 재시작

### 여정 B. 단발 작업판 실행 (이미지 한 장 / 텍스트 한 번)

1. 작업판 목록 (전역) → 작업판 클릭
2. 입력 양식 채우기 → 실행
3. 결과 확인 → 다른 작업판으로 결과 이어가기 (다음 작업)

### 여정 C. 컨텐츠 관리 (이미지 / 영상 / 텍스트)

1. 사이드바 \"내 컨텐츠\" → 탭 (생성된 이미지 / 업로드된 이미지 / 영상 / 직접 작성 텍스트 / 생성된 텍스트)
2. 검색 / 필터 / 태그 / 즐겨찾기 ...
3. 대화 히스토리에서 마음에 드는 응답을 \"저장\" 으로 영구 보관

### 여정 D. 관리자 작업 (admin only)

1. 작업판 정의 (customField 설계, 모델 화이트리스트, 그룹 권한)
2. 서버 등록 / 모델 동기화 (ComfyUI / SaaS)
3. 사용자 / 그룹 관리

## 4. 기술 / 제약 / 톤

- **기술 스택**: React 18 + Material-UI v5 + React Query + react-hook-form + react-dropzone + Bull queue
- **언어**: 한국어 (영문 기술용어 일부 혼합)
- **테마**: 현재 light 단일 (다크모드 미지원). 다크모드 도입 여부는 별도 결정
- **반응형**: 모바일 (xs ~360px), 태블릿 (sm ~600px), 데스크탑 (md ~960px+) 모두 지원. **xs / md 만 사용 중** — sm / lg / xl 거의 미사용 → 단순화 권장
- **톤**: \"화려함\" 보다 \"명확함\". internal tool 답게 정보 위계가 잘 보이고 작업 흐름이 매끄러워야 함. 인스타그램형 X, Notion / Linear 같은 정돈된 productivity 톤 ✓.
- **금지**: 과도한 애니메이션, 화려한 색, 정보 가리는 모달 남발, 페이지 사이 일관성 깨는 디자인 변화.

## 5. 현재 디자인 상태 / 한계

상세는 [TOKENS.md](./TOKENS.md) 참고. 요점:

- MUI 기본 + hardcoded hex 색 혼재 (특히 AppBar / Sidebar 의 `#2c3e50` / `#34495e` 시리즈)
- 한 컴포넌트 안에서 spacing / borderRadius / button size 가 임의 — 명문화된 디자인 시스템 부재
- 모바일 레이아웃이 다음 단계 우선순위 (현재도 동작은 하지만 정돈도 떨어짐)
- 정보 위계가 약함 — Card / Paper 사용은 많지만 중요도 표현 부족

## 6. 산출물 요청 (디자인 에이전트가 만들 것)

### Phase 1 — 디자인 시스템 토큰 (코드 적용 가능한 형태)
- 컬러 팔레트 (primary / secondary / neutral / status / navbar 같은 surface scale 정리). MUI palette 객체 형태로.
- 타이포 스케일 (h1~h6, body1/2, caption). 한국어 가독성 고려한 line-height / letter-spacing 권장.
- 간격 단위 (4 / 8 / 12 / 16 / 24 / 32...) — 명문화. 컴포넌트별 권장 padding / margin.
- 둥글기 단위 (4 / 8 / 12...).
- Shadow / Elevation 단계 정의 + 사용 가이드 (depth 표현 vs flat surface).

### Phase 2 — 핵심 컴포넌트 변형 (Card / Chip / Button / Tabs / Stepper)
- 우리 톤에 맞춘 커스텀 (특히 모바일 컴팩트 모드 강조).
- 상태 (default / hover / active / disabled / loading) 모두 명시.
- 한국어 라벨 길이 (예: \"대화 이어가기\", \"파이프라인 단계 추가\") 가 안 깨지는지.

### Phase 3 — 핵심 3개 페이지 visual mockup (wireframe + mockup 둘 다)
1. **프로젝트 상세 페이지** (탭 구조 + 파이프라인 / 세계관 / 히스토리). 데스크탑 + 모바일.
2. **파이프라인 빌더** (단계 리스트, 단계별 사전 입력 / 문서 연결, 드래그 reorder). 데스크탑 + 모바일.
3. **파이프라인 실행 + 히스토리** (Stepper, 진행 상태, 결과 미리보기, 재시작). 데스크탑 + 모바일.

## 7. 우선순위 / 비범위

- **In scope (이 라운드)**: 위 Phase 1 + Phase 2 + Phase 3 (핵심 3개 페이지).
- **Out of scope (다음 라운드)**: 다크모드, 관리자 페이지 (admin / 모델 관리), 인증 페이지, 대시보드 위젯 다양화.
- 점진 적용 — 한 번에 다 갈아엎지 않고 토큰 → 핵심 3개 페이지 → 사용자 피드백 → 다음.

## 8. 첨부 (디자인 에이전트에 함께 전달)

- [TOKENS.md](./TOKENS.md) — 현재 코드 기준 디자인 토큰 추출
- 현재 화면 스크린샷 (사용자가 직접 캡처해 전달 — 데스크탑 / 모바일 각 핵심 3개 페이지)
- (선택) 참고할 유사 도구 디자인 (Notion / Linear / Vercel dashboard / etc.) — 사용자가 톤 정하실 때 같이 첨부

## 9. 결과물 형태 요청

- Figma 파일 (페이지 / 컴포넌트 / 토큰 분리) — 가장 선호
- 또는 PNG mockup + 토큰 JSON / Markdown — 코드 적용 시 그대로 옮길 수 있게
- 토큰은 MUI ThemeOptions 형태로 코드 스니펫 함께 주면 적용 즉시 가능
