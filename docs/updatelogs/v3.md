# v3 업데이트 내역

## v3.12.0

### 새 기능 (관리자)
- **프로젝트 내보내기 / 가져오기가 추가되었습니다.** 프로젝트 상세의 "내보내기"로 프로젝트의 작업판·세계관 문서·파이프라인·태그를 파일 하나(JSON)로 저장하고, 프로젝트 목록의 "가져오기"로 그 파일에서 새 프로젝트를 통째로 만들 수 있습니다. 다른 환경으로 옮기거나, 잘 만든 프로젝트를 템플릿처럼 복제할 때 쓰세요.
- 가져올 때 작업판이 연결될 서버는 자동으로 맞춰지고, 애매하면 직접 고르는 단계가 나옵니다.
- 새 프로젝트 만들기 창에도 "내보내기 파일에서 시작" 버튼이 생겼습니다.

자세한 개발 변경 목록은 GitHub issues 의 `v3.12.0` label 검색 참고.

## v3.11.0

### 작업판 편집기 전면 개편 (관리자)
- **탭을 오가던 편집기가 한 화면으로 바뀌었습니다.** 왼쪽에 기본 정보·권한·워크플로우 요약, 가운데에 입력 양식, 오른쪽에 선택한 필드의 상세 설정과 사용자 시점 미리보기가 함께 보입니다.
- **저장하지 않고 나가면 확인을 묻습니다.** 편집 중에는 상단에 "저장되지 않은 변경" 표시가 떠서 잃어버릴 걱정 없이 작업할 수 있습니다.
- **서버를 바꿀 때 워크플로우가 지워질 상황이면 미리 경고합니다.** (기존에는 경고 없이 사라졌습니다)
- **이름을 채우지 않은 입력 필드는 저장 시 조용히 사라지는 대신, 안내와 함께 해당 필드로 이동합니다.**
- **워크플로우 편집이 전용 화면으로 커졌습니다.** 치환 변수 목록이 사용 중/미사용으로 표시되고, 클릭하면 커서 위치에 바로 들어갑니다. 저장 전에 JSON 유효성도 검사할 수 있습니다.
- 필수 입력 강제, 체크박스·숫자 필드 표시 등 편집기에서 정한 것이 실행 화면에 그대로 반영되도록 내부 구조를 통일했습니다 (미리보기 = 실제 화면).

자세한 개발 변경 목록은 GitHub issues 의 `v3.11.0` label 검색 참고.

## v3.9.0

### 안정성 개선
- **서버·작업판·프로젝트·태그를 만들거나 수정할 때 입력값 검사가 강화되었습니다.** 잘못된 형식의 값이 들어오면 저장 전에 명확한 안내 메시지로 알려줍니다. 정상적인 사용에는 달라지는 것이 없습니다.

## v3.8.17

### 안정성 / 정리
- **작업 화면과 관리자 화면에서 브라우저 개발자 콘솔에 입력 내용이 출력되던 것을 제거**했습니다. 이미지 생성·작업판 편집 시 입력한 값들이 콘솔에 남지 않습니다.
- 예상치 못한 서버 오류가 났을 때 **내부 오류 문구가 화면에 그대로 노출되지 않도록** 다듬었습니다.
- 내부 색상 정리로 대시보드 통계 그래프 선 색이 테마와 어울리는 보라 톤으로 통일되었습니다 (다크 모드에서도 자연스럽게 보입니다).

## v3.8.16

### 보안 / 안정성
- **내부 구성 요소의 보안 취약점들을 최신 패치로 해소했습니다.** 파일 업로드·메일 발송 등에 쓰이는 구성 요소들을 안전한 버전으로 업데이트했습니다. 사용법이나 화면 변화는 없습니다.
- 로그인 과정에서 **브라우저 개발자 콘솔에 계정 정보 일부가 표시되던 것을 제거**했습니다.

자세한 개발 변경 목록은 GitHub issues 의 `v3.8.16` label 검색 참고.

- 작업 히스토리의 텍스트 결과 상자 좌우 여백을 넉넉하게 다듬었습니다 (#674).

## v3.8.8

### 복원 기능 개선
- **복원한 데이터의 생성 시간이 원래대로 보존됩니다** (#646). 이전에는 복원하면 모든 작업·콘텐츠의 생성 시간이 복원한 시점으로 바뀌었는데, 이제 백업에 담긴 원래 시간이 그대로 유지됩니다.
- **새 서버로 이전할 때 쓰는 "완전 교체" 복원 모드가 추가되었습니다** (#646). 빈 서버에 복원할 때 이 옵션을 켜면 기존 데이터를 깨끗이 비우고 백업으로 통째 교체합니다. 새로 만든 관리자 계정과 백업 속 계정이 충돌해 내 콘텐츠·작업 히스토리가 보이지 않던 문제가 해결됩니다. (복원이 끝나면 백업에 담긴 계정으로 다시 로그인하세요.)
- **복원 진행 상황이 전용 화면으로 표시되고, 완료되면 자동으로 새로고침됩니다** (#650). 복원 중에는 단계별 진행률이 보이고, 끝나면 깨끗한 상태로 이어집니다.
- **복원 중 "작업을 찾을 수 없습니다" 오류가 계속 뜨던 문제를 고쳤습니다** (#648). 복원은 정상 진행되는데도 화면에 오류가 반복 표시되던 것을 바로잡았습니다.
- 복원이 끝나면 이전 작업 큐와 캐시를 정리해, 복원한 데이터와 어긋나지 않도록 했습니다.

## v3.8.0

### ComfyUI 워크플로로 작업판 만들기
- **ComfyUI 워크플로를 붙여넣으면 작업판 초안이 자동으로 만들어집니다.** 작업판 화면의 "워크플로 변환"에서 ComfyUI의 "Save (API Format)"으로 저장한 워크플로 JSON을 넣으면, 프롬프트·모델·시드·크기·LoRA 등 필요한 입력이 자동으로 변수로 추출되어 작업판 초안이 생성되고 편집기로 이어집니다.
- **이 워크플로에 필요한데 서버에 없는 커스텀 노드를 알려줍니다.** 어떤 노드가 빠졌는지, 어디서(어느 저장소) 설치하면 되는지 함께 안내합니다.

### 버그 수정
- **백업이 중간에 멈췄을 때 계속 "진행중"으로 남던 문제를 고쳤습니다** (#620). 디스크 부족 등으로 백업이 중단된 경우, 서버를 다시 시작하면 멈춰 있던 작업이 자동으로 정리됩니다.

### 다크모드 (Phase 7)
- `buildVccTheme(mode)` 함수 — light/dark palette 양쪽. 디자이너 핸드오프 토큰 (dark: 배경 #0E1015, Paper #161A22, primary #7676E0, text #E8E9EE 등) 적용. hardcoded hex (navbar/divider/outlined notchedOutline) → theme.palette 참조 함수형 styleOverrides 전환.
- `ColorSchemeContext` — mode (`light`/`dark`/`system`), matchMedia(prefers-color-scheme) 구독, localStorage 영속, html `data-theme` attribute 동기화.
- **디바이스별 영속** — 백엔드 user preferences 동기화 안 함. 기기마다 모드 선호 다를 수 있다는 사용자 정책 반영.
